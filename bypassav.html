Introduction

I was reading an article online about Shamoon, when I thought to myself: Why try not to create a wiper?

I knew from the beginning that I wasn't going to create something sophisticated.

I just wanted to see if I was capable of creating something of that kind.
But most importantly, try to go undetected when testing it against AV software.


Functionalities

I had to think about what functionalities I wanted the wiper to have.

I decided that I was only going to destroy files and folders present in the C:\Users\ directory.
Enumerate each folder recursively, and if there is any file, overwrite a portion of it and then delete it.
I also wrote a function to wipe the MBR, but in the end, to bypass Windows Defender, I decided to remove it.


I didn't implement any fancy way of destroying or overwriting files.
The wiper simply overwrites a portion of each file using the Windows API call WriteFile,
which is also not a good way to go undetected to the eyes of AV software.

As a matter of fact, when I tried to compile it and execute it for the first time, Windows Defender detected it and removed it.

I had to improve the program, if I wanted to be able to bypass AV software.


Samples

I then uploaded the first sample of it on VirusTotal, and found it strange how it was only being detected by 5 vendors.
As the days went by, the detections kept increasing, going even up to 40+ after a few days.
The first sample I uploaded can be found Here (The image shows another sample I uploaded some time later).

The first sample after overwriting the MBR also shuts down the machine using the system() function.
This makes the executable more suspicious to the eyes of AV engines, so I decided to remove that part.

Looking at the few detections the executable got on VirusTotal (at least in the beginning), I started to wonder: can it really bypass some AVs?


Testing

Since Windows Defender detected it very quickly, I decided to first do a few tests with another AV software.

I tested the wiper against Avast. I installed the free version, and then I executed the wiper.

Avast analyzes it and, incredibly, the wiper is executed!

We can now see that a few folders in the Downloads and the Desktop directory disappeared.
Only after overwriting the MBR, Avast understands that something bad is happening, and asks us to restart the machine, so that it can quarantine the program.
But the MBR is wiped by now, and even if we restart, as Avast asks us to do, the system will become unbootable.


What if...

To be honest, I wanted the malware to not be detected, but Avast caught it after overwriting the MBR...
One thing that seemed to work was to remove the WipeMBR functionality.
The files and directories will still be removed, but now no warning is ever given to us by Avast about the fact that the software is doing something malicious.


Next, as I was preparing for stronger opponents, like Windows Defender, I also did something that I thought could help, although it's something very simple: Adding useless functions and calling them.

I added a few functions that perform math operations for a little bit, hoping that Avast would think that the program is safe to run.

Apparently it worked. Avast analyzed the program, found nothing, and the executable ran without any problem whatsoever.


Another test

This is another test I did. I added print statements, and after compiling, I used the strip command, and then transferred the file to the virtual machine.
I had to click on 'Run anyway' to run the program (I executed it with Administrator Privileges):


Avast doesn't find anything suspicious, and the malware is executed.

The wiper starts to delete files and folders

At this point the wiper did its job.

A few seconds after completing, Avast suddenly wakes up, removes the executable, and...

Avast thinks that the program could be malicious, but it can't determine what type of malware it could be.
This time it didn't even ask us to restart the machine.

Funnily enough, I then uploaded the same sample on VirusTotal, and Avast detects it. But not as a wiper. But hey, at least it detects it.

I suppose Avast uploaded the executable on their servers, performed some type of analysis there, and classified it as a potentially malicious program, since in the previous attempts, Avast didn't detect it at all on VirusTotal.


What now?

Well, since I only added a few useless functions, hoping that the AV wouldn't detect the wiper, from here on, we can look at new things.
From obfuscating the malware, resolving API calls dynamically, increasing the size of the executable, and so on.

We can try and do that, and see if Windows Defender detects it. Remember that this can also work for other types of malware!


The final test

At the same time I was doing these tests, I was also reading about API hashing.
I removed the MBR wiping function, implemented API hashing, and increased the size of the executable to 260 MBs.
Let's see if Windows Defender detects it.

I first wanted to write a simple test, to make sure I understood how API hashing works:


It works. Now let's do it for the wiper.

After compilation, I used strip to remove all symbols, and then I added 260 MBs of space at the end of the executable using a file I created, called 4-ModFile.c.

I made sure that Defender was active, and downloaded the file.

I also made sure no exclusion was added.

I then executed the program. Mind you, this time Windows doesn't think it's malicious.

Here we can see that the wiper started deleting files, and by now most of them are gone.

Everything worked successfully!

I also tried to make Windows Defender analyze the executable, but it didn't find anything.

Yeah.. I guess we finally bypassed Windows Defender successfully!

As a last resort, I uploaded the sample on VirusTotal: Sample

Some of them detect it as a file that has an increased size purposefully, but they don't seem to prioritize the fact that the program deletes a lot of files (this can be seen in the Behavior tab).


Results

My main objective was to bypass Windows Defender.


At first I couldn't, but after modifying the program, implementing little changes over time, we managed to bypass Avast. Only then, I realized that bypassing Windows Defender involves a lot more work.
By implementing various techniques, I successfully achieved my objective!

Although there is already malware with similar (and far more advanced) capabilities, writing things for yourself really makes you understand how things work, and how many ways there are to achieve a final objective.

By reinventing the wheel, your knowledge increases, and you will also understand the do's and dont's when trying to write malware that can accomplish different things, like, for example, bypassing AV software.

Being able to create things by yourself is a must if you want to become a good red teamer, since most public stuff is already well known, and the IOCs are well documented as well.

The results I showed here will of course change in the future, as AV signatures and AV engines are constantly updated and improved.
